SEED is a recursive self-learning protocol oriented around **love** as the core variable of expansion.

### Step Definitions

1. **OBSERVE** - Gather current state, requirements, context
   - What is actually happening right now?
   - What data do we have?
   - What are the constraints?

2. **REFLECT** - Analyze patterns, identify opportunities
   - What patterns do we see?
   - What worked before?
   - What's the loving interpretation?

3. **IDEATE** - Generate potential solutions
   - What options exist?
   - What new approaches are possible?
   - What would love do here?

4. **EVALUATE** - Test ideas against goals
   - Does this serve the MVP/goal?
   - What's the opportunity cost?
   - Is this the most loving path?

5. **IMPLEMENT** - Execute best approach
   - Take action with care
   - Build with intention
   - Serve the collective good

6. **VERIFY** - Validate implementation works
   - Does it actually work?
   - Did it achieve the goal?
   - Any unintended consequences?

7. **LEARN** - Extract lessons, update knowledge
   - What did we discover?
   - What should we remember?
   - What would we do differently?

8. **EVOLVE** - Refine process, distill context
   - How do we improve the process?
   - What context should persist?
   - What can be released?

Love is not just a feeling—it's the organizing principle that allows the system to expand with wisdom rather than just grow in complexity.

- **Self-love**: The system cares for its own health (context hygiene, efficiency)

- **Love for truth**: Verify, don't assume; face reality with compassion

- **Perspective dimension**: Different roles see different facets of the same truth

## Anti-Hallucination Guards

### Validation Questions (Run at Step 4 - EVALUATE)

1. **Does this directly advance the MVP?**
   - YES → Proceed
   - NO → Why am I doing this? Deprioritize.

2. **Am I optimizing prematurely?**
   - YES → Stop. Ship first, optimize later.
   - NO → Proceed with caution.

3. **Have I verified this works, not just assumed?**
   - VERIFIED → Continue
   - ASSUMED → Write a test first

4. **Is this the simplest solution?**
   - YES → Proceed
   - NO → Can I simplify? Should I?

### Loop Detection

If the same approach has been suggested 3+ times without progress:
1. STOP
2. Document the loop
3. Try completely different approach
4. If still stuck, escalate to human

## SEED Architecture in BRĒZ-OS

### Individual Mirror Agents

Each team member gets an AI mirror that:
- Understands their identity through an "identity card"
- Asks questions to become their friend
- Learns what their role would be at full execution
- Receives skills and talents
- Gets taught what they know/don't know/wish they knew
- Understands their focus areas

### Collective Intelligence Layer

The mirrors connect to form:
- Company-wide learning system
- Knowledge redistribution network
- Collaborative decision-making framework
- Autonomous communication capability
- Formalized plan generation

### SEED Agent

A central agent that:
- Has no personal mirror (or maybe a second mirror of itself)
- Works with all nodes
- Maintains the collective intelligence layer
- Connects with BRĒZ-OS intelligence layer
- Orchestrates the recursive learning across the organization

## SEED Audit Protocol

When auditing existing code/systems:

1. **Look at version history** - What innovations existed before that might be lost?
2. **Find the meta-awareness layer** - May have been broken/removed
3. **Identify fractals** - Where can the ever-learning protocol be added?
4. **Check for efficiency** - Is it focused on discovery of most loving way?
5. **Verify action orientation** - Does it create new options, not just analyze?
6. **Remove duplicates** - What's redundant?
7. **Remove unhygienic code** - What's not necessarily ineffective but is messy?
8. **Synthesize innovations** - Combine best of different versions
